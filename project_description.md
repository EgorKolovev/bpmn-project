# Название проекта

Проектирование BPMN-диаграмм на основе описания с использованием LLM

# Цель проекта

В компаниях и организациях существует необходимость формализации бизнес-процессов. Один из самых распространенных стандартов для этого — BPMN (Business Process Model and Notation). Традиционно проектирование диаграмм требует ручной работы аналитиков: они собирают требования, читают описания процессов и переносят их в нотацию BPMN. Это долго, подвержено ошибкам и требует высокой квалификации.

Современные большие языковые модели (LLM) способны анализировать текстовые описания процессов и автоматически преобразовывать их в формализованные структуры. Цель проекта — исследовать и разработать подход, который позволит автоматически строить BPMN-диаграммы на основе текстового описания процесса с использованием LLM.

Команда изучит существующие статьи, библиотеки и практики применения LLM для структурирования бизнес-процессов, а затем создаст прототип системы, которая принимает описание бизнес-процесса и выдает BPMN-диаграмму (например, в формате XML BPMN 2.0 или визуальной схемы).

# Результат (продукт)

Сервис, принимающий текстовое описание бизнес-процесса и автоматически генерирующий BPMN-диаграмму.

Прототип должен уметь генерировать результат и экспортировать его в BPMN 2.0 XML для загрузки в редакторы (например, Camunda Modeler, Макросы Confluence и т.д).

# Критерии приемки

## **Требование к результату (продукту)**

1. Реализована система преобразования текстового описания в BPMN-диаграмму.

2. Поддерживается экспорт в формат BPMN 2.0 (XML).

3. Проведен сравнительный анализ нескольких LLM-подходов (prompt engineering, использование специализированных библиотек, fine-tuning).

4. Реализован кастомный Web-интерфейс на Angular 2 в виде чата для пользовательского взаимодействия

5. Подготовлен отчет с описанием методологии, сравнением подходов и рекомендациями.

## **Требования к технологиям**

- LLM (OpenAI, HuggingFace, LM Studio и др.)

- Camunda Modeler (для проверки)
